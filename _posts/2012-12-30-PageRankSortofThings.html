起因很简单,

就是我自己造了一个词,并且把它同时放到了这个博客和新浪微博里,如果有人在我的新浪微博留意到这个词并且google之,那么希望她能找到这里,正好结合我们课上学到的一些知识,分享一下这些小把戏


几个概念:

Boolean Model,就是一张巨大无比的表格(课表...),竖栏是n多关键词,比如"家乐福","大陆","欧美","电影","动漫",横栏则是N多个pages,页面.如果一个页面包含某个关键词(这一行,这一列的交点表示他俩之间的关系),这个交点就会标记为1,不包含则标记为0. 当你在搜索引擎中输入几个关键词,甚至一句话(最终这句话也要被分解成几个关键词),引擎就会查表,利用或且非(And,Or,Not)的关系进行布尔运算,得出结果就是你得到的页面.据个例子:有五个网页D1至D5.

D1 D2 D3 D4 D5
家乐福 1 0 0 1 1
大陆 0 0 0 1 1
欧美 1 1 0 1 1
电影 0 1 0 1 0
动漫 0 1 0 0 1

对于上述5个名词,如果我们搜索"欧美""电影"非"动漫"",引擎会列出包含上述关键词的三行,并且由于对"动漫"的定义为"非",对这一行的01序列取反,将三行01序列运算后得到的01序列,00010,也就是D4这个页面是符合的,将他呈献给用户.

Crawler,爬虫机器人

刚才所说的那个表是怎样建立起来的呢,又是怎样更新和维护的呢?要靠Crawlers了,其实有关爬虫机器人,或者叫web spider的运作机理很多地方都有介绍,各个搜索引擎部署策略不同,甚至是google自己在不同时期部署crawler的策略也不同,简单理解下吧,Crawlers被分发,蔓延到各个网页,搜集,分析网页,将结果回传给引擎.这里只是简单提一下,很多东西Crawler是不能分析的,比如一个按钮上的文字,通常不会作为关键词被记录进去,Javascript脚本里的很多东西也不容易被记录

Page Rank

学这么多年计算机,用这么多年网络,居然才知道page rank不是个朴素的英语单词,居然就是Larry Page,拉里佩奇,谷歌创始人之一, 用自己的lastname给起的名字,(佩奇排名,多别扭啊).传统的网页排名机制是对单一网页进行分析,而PageRank则更注重页面之间的关系,页面之间的链接,links作为参考依据.举个例子,pagerank以前,你的某个网页里面暗藏100个某个女生的名字,我的页面里她只出现过一次(剧情需要,实际上一次也没出现过:()那么你的网页排名一定比我高.但是在pagerank机制中,你的网页只出现在你自己的网站里,不与外界沟通,而我将这个网页连接到我的新浪微博,百度空间,google+,facebook,维基百科上(够屌丝吧...),这个连接可以单项也可以双向,比如我在我的blog里加入到了跳转到谷歌搜索,和跳转到我的新浪微博这个连接,就是主动"吐出"了链接,我在我的百度空间里发一篇文章,里面的内容包含我这个博客里含有那个女生名字的页面的连接,就相当于"吃进"了一个连接.Crawlers经过探索,挖掘,记录这种连接关系,对比你的链接关系(你只在你自己的页面link,既不吐出,也不吃进连接),那么我一定比你高.

当然,实际的rank机制要比这个复杂很多,而且你也可以显式的(比如在HTML里用一个tag)告诉Crawler不要记录这个关键词(这个名字...),甚至推荐某个关键词(我是学霸),不过至于谷歌听不听你的劝告,那就不一定了.(比如他可以不记录这个女生的名字,但是也不相信我是学霸,擦,被看穿了).
不仅仅是数量,链接的质量也很影响page rank,你和全班50个人的"个人blog"相互链接,我和谁都没有链接,但是有一条Amazon的出站链接指向了我,那么我比你们都要高很多很多.(不过也保不齐我的同学们谁会成为下一个马云,柳传志).综合数量和质量这两点评价links关系,就好比有向连通图,每个点都是一个网页(不是网站!).假设咱俩的网站以及里面的网页的基础的pagerank排名都一样,我们主页的pagerank因子都是2.都没有吃入的连接.现在我们同时加入一个连接到谷歌的超链接.因为我们吐出了一个链接,我们的pagerank基础因子都下降了(我们会把我们下降的部分送给谷歌,同理我之前说如果Amazon主动连接到了我,我吃进他的链接,rank因子会上升,亚马逊实在太牛了,所以尽管给我一个链接我的因子可能就上千上万了)注意了,你的blog只有两个页面,一个主页,一个包含所有文章全文的页面,仅主页包含指向Google的连接.我的blog有10个页面,一个主页,9篇文章,每篇文章可以返回到主页,甚至有的文章能跳到其他文章,但也仅是主页包含指向Google的连接.那么此时,你的主页rank因子下降的很长多,可能变为1到1.3之间的一个数,但是我主页的rank因子最多下降为1.8 因为这个主页只把它因子的1/10吐出给谷歌,而你要把你的1/2吐出给谷歌.所以有了1和1.8这两个数字.但由于这个变动,咱们各自网站(网页的集合)的因子会发生变化,经过复杂的动态的权值调整以后,最后的因子要略高,毕竟子网页还有连接到主页的连接


以上这些算是自己对于搜索引擎.pagerank的一点小小的认识,并不十分严谨(比如因子,google搜索引擎里所有页面的因子综合应该是1),更没有给出计算公式,希望抛砖引玉,有兴趣的朋友可以看看"Google's PageRank and Beyound-The Science of Search Engine Rankings,这本书.
